# Accessibility

iPad accessibility includes the iPhone baseline plus keyboard, pointer, focus, and multi-window concerns.

## Dynamic Type

- Use text styles and flexible layout.
- Test at extra-extra-extra-large and an accessibility size.
- Let sidebars collapse or reflow before text clips.
- Avoid fixed-height rows with multi-line localized text.

## Focus And Keyboard

- Keyboard focus must be visible.
- Focus order should follow visual hierarchy.
- Shortcut discoverability matters for repeated actions.
- Selection state must not rely on color alone.

## Pointer

- Hover and pointer effects should clarify, not distract.
- Custom controls need clear pointer affordance.
- Dense controls need enough spacing for touch and pointer use.

## VoiceOver

- Group complex rows.
- Label icon-only toolbar buttons.
- Hide decorative imagery.
- Provide meaningful rotor/order behavior for timelines and grids.

## Contrast

- Body text: at least 4.5:1.
- Large text/icons: at least 3:1.
- Panel separators and selection states must remain visible in dark mode and increased contrast.
