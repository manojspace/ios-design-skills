# Input Systems

iPad users move between touch, pointer, keyboard, Pencil, and drag/drop. Design the same workflow for multiple inputs.

## Touch

- Hit targets remain at least 44x44pt.
- Gestures need visible alternatives.
- Important commands should not require hidden gestures.

## Pointer And Trackpad

- Add hover states where they clarify interactivity.
- Use pointer effects for custom controls.
- Make dense tables, grids, and toolbars easy to target.
- Preserve touch ergonomics; pointer support should enhance, not replace, touch.

## Hardware Keyboard

Add shortcuts for repeated actions:

- New/create.
- Search.
- Save/done.
- Delete/archive.
- Navigate next/previous.
- Toggle sidebar/inspector.
- Open command/menu surfaces.

Ensure focus order and visible selection are clear.

## Apple Pencil

Use Pencil when it has a job:

- Annotation.
- Sketching.
- Handwritten notes.
- Precise selection.
- Markup.

Do not add Pencil-only features for core tasks unless the app is a drawing, note, or creative tool.

## Drag And Drop

Support drag/drop for objects users naturally move:

- Trips, itinerary items, tasks, files, images, documents, events.
- Reordering and grouping.
- Importing from Files, Mail, Calendar, Photos, or Safari.

Always provide non-drag alternatives.
