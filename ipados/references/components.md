# iPad Components

Choose components that use iPad space productively.

## Sidebar + Content

Use for projects, trips, folders, collections, documents, settings, or account-scoped apps.

Requirements:

- Clear selected state.
- Collapsible behavior in compact windows.
- Search/filter close to the collection.
- Keyboard navigation and shortcut access.

## Content + Inspector

Use when editing or reviewing an object with properties, metadata, filters, history, or related actions.

Inspector rules:

- Secondary to the primary content.
- Resizable/collapsible when possible.
- No duplicate primary action clutter.

## Dashboard

Use large canvas for context:

- Primary status.
- Next action.
- Timely alerts.
- Recently changed objects.
- Comparison widgets.

Avoid magazine-like decoration unless the app is editorial.

## Timeline / Itinerary

For travel and schedule products:

- Day/date grouping.
- Local time and time zone clarity.
- Side-by-side map/detail when useful.
- Drag/reorder for planning.
- Document/reservation attachment surfaces.

## Dense Table Or Grid

Use for scannable collections:

- Adaptive columns.
- Sort and filter.
- Multi-select.
- Context menu.
- Keyboard navigation.

## Popover

Use for lightweight choices, filters, action menus, date pickers, and object creation shortcuts. Do not push a full-screen sheet for a tiny task.
