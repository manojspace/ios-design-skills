# Navigation, Windows, And Multitasking

iPadOS is a windowed, multitasking platform. A good iPad app survives more than full-screen landscape.

## Navigation

- Use split navigation when users benefit from persistent context.
- Use sidebars for major destinations, collections, accounts, projects, or trips.
- Use toolbars for common actions and mode changes.
- Use inspectors for secondary details, not another push when side-by-side editing is better.
- Keep tab bars only when they remain the clearest mental model.

## Windows And Stage Manager

Design for:

- Full screen.
- Split View / half screen.
- Stage Manager resizable windows.
- Multiple windows/scenes when the workflow benefits from comparing documents, trips, tasks, or records.

Window checks:

- Content reflows instead of clipping.
- Primary actions remain reachable.
- Empty spaces are intentional.
- No fixed iPhone-width assumptions.

## Menu Bar And Commands

For productivity-style iPad apps, expose repeated actions through:

- Menu bar commands.
- Keyboard shortcuts.
- Context menus.
- Toolbar items.

Use clear command names and organize by task: File, Edit, View, Navigate, Format, Window, Help, or domain-specific menus when appropriate.

## Sheets And Popovers

- Use popovers for lightweight contextual choices.
- Use sheets for focused flows.
- Avoid full-screen modal flows unless the task truly needs immersion.
- On iPad, a huge modal with one field is usually a design failure.
